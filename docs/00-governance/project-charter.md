# Project Charter

## Project

- **Name:** KPM — Kozen Project Management
- **Organization:** Kozen
- **Current phase:** Phase 1 — Requirement preparation

## Goal

Build a project management product that fits Kozen's real workflows, can be launched reliably, and is supported by clear product, technical, and delivery documentation.

## Guiding principles

1. Requirements before architecture
2. Product clarity before code volume
3. Java-first technical thinking, but not Java-only dogma
4. Maintainability, availability, and usability are first-class requirements
5. Every major decision should be documented

## Delivery phases

| Phase | Output |
| --- | --- |
| Phase 1 | Requirement intake, PRD, product prototype |
| Phase 2 | Technical solution |
| Phase 3 | Working product |
| Phase 4 | Deployment package, acceptance support, release readiness |
