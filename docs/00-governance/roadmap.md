# Product Roadmap

## V1 — 可试点版

目标：让一部分团队可以开始真实使用 KPM，至少能统一查看项目、流程、阶段和责任人。

### Included

- 用户管理
- 角色管理
- 部门管理
- 权限管理
- 项目管理
- 流程模板管理
- 项目成员配置
- 项目阶段展示
- 阶段负责人 / 负责部门配置
- 项目客户列表与客户状态维护
- 客户需求列表与需求纵览
- 需求与任务联动
- 阶段详情、阶段资料库、阶段留言
- 项目资料区与阶段资料共享
- 轻量任务管理
- 任务分类
- 任务详情
- 阶段卡点求助
- 中英文双语兼容

### Product value

- 解决“项目当前走到哪里”
- 解决“每个阶段由谁负责、当前各阶段分别是什么状态”
- 解决“新人不知道该找谁”
- 为后续复杂协作能力建立统一数据基础

## V2 — 协同增强版

目标：开始解决跨部门协作过程中的真实摩擦。

### Candidate scope

- 阶段资料与信息共享
- 阶段预计完成时间
- 单独延期申请与延期原因
- 审批流
- 基础通知中心
- 基础看板 / 风险视图

## V3 — 集成与规模化版

目标：把 KPM 接入现有工作生态，并提升管理效率。

### Candidate scope

- Jira 集成
- 钉钉通知
- 更细粒度权限控制
- 报表与管理驾驶舱
- 审计日志
- 模板复用和流程分析

## Phase 1 — Requirement preparation

- Gather business goals, users, workflows, permissions, and constraints
- Clarify MVP scope versus later releases
- Produce PRD
- Produce and review product prototype

## Phase 2 — Technical solution

- Define architecture
- Select stack
- Define data model, security model, API style, deployment topology, and non-functional requirements

## Phase 3 — Development

- Build incrementally
- Verify usability, performance, availability, and correctness
- Keep implementation aligned with the approved PRD and prototype

## Phase 4 — Acceptance and launch

- Prepare deployment instructions
- Prepare one-command deployment where practical
- Support testing, bug fixing, and release readiness
