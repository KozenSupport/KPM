## 1. 测试范围

### 1.1 本轮测试覆盖

本轮重点验证以下内容：

1. 登录、退出、基础导航。
2. 项目管理、项目详情、阶段详情。
3. 客户管理、客户详情、联系人、资料、跟进记录。
4. 任务管理、任务详情、任务状态、附件、评论。
5. 订单管理、订单修改记录、订单与客户/项目联动。
6. 资源管理，包括用户、部门、角色、权限、枚举配置。
7. 统计看板，包括订单统计、资源分配、技术支持情况、客户活跃度。
8. 表单校验、二次确认、弹窗交互、长列表折叠展示。
9. 前后端数据联通：页面操作是否能调用接口并落库，刷新后数据是否仍存在。
10. 基础兼容性和易用性检查。

### 1.2 本轮暂不作为通过门槛

以下内容可记录问题或建议，但不建议作为本轮阻塞项：

1. 真实 1000 并发压测。
2. 真实地图 SDK 加载效果。
3. 审批流程。
4. 钉钉/JIRA/CRM 的真实第三方集成。
5. 生产级权限拦截与安全审计。
6. 多浏览器深度兼容测试。

## 3. 缺陷等级建议

| 等级 | 定义 | 示例 |
|---|---|---|
| Blocker | 阻塞核心流程，无法继续测试 | 登录失败、项目详情无法打开、接口全部不可用 |
| Critical | 核心业务错误或数据丢失 | 新增订单未落库、任务状态联动需求错误、删除无二次确认 |
| Major | 重要功能异常，但有绕过方式 | 某个筛选条件无效、弹窗关闭后状态异常 |
| Minor | 体验或展示问题 | 文案不准确、间距不佳、提示不够明确 |
| Suggestion | 优化建议 | 增加快捷筛选、增加批量操作 |

## 22. 通过标准建议

建议满足以下条件后，认为当前版本可以进入更正式的试点或下一阶段：

1. 所有 P0 用例通过。
2. P1 用例通过率不低于 90%。
3. 不存在 Blocker 或 Critical 缺陷。
4. Major 缺陷都有明确修复计划或可接受绕过方式。
5. 新增、修改、删除类操作均完成二次确认和必填校验。
6. 关键资源配置能够实际联动业务页面。
7. 刷新页面后关键新增/修改数据仍然存在。
