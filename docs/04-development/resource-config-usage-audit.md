# KPM 资源配置联动检查

> 更新时间：2026-06-01。本文件用于跟踪“资源管理中配置的数据，是否能在业务页面实际使用”。

## 已检查并联动的配置项

| 资源配置项 | 后端枚举类型/数据源 | 业务使用点 | 当前处理 |
| --- | --- | --- | --- |
| 任务状态 | `task_status` | 任务列表筛选、任务创建默认状态、任务详情状态下拉、任务状态流转配置 | 任务详情状态下拉已改为展示所有启用状态，并显示流转建议 |
| 任务状态流转 | `kpm_task_status_transitions` | 资源管理 / 任务状态配置、任务详情状态说明 | 用于提示推荐流转；不再错误地限制任务详情只能选部分状态 |
| 任务分类 | `task_category` | 新建任务、阶段任务、卡点任务、任务列表筛选 | 从资源 bootstrap 动态加载 |
| 优先级 | `priority` | 新建任务、阶段任务、卡点任务、任务详情、需求优先级 | 从资源 bootstrap 动态加载 |
| 阶段状态 | `stage_status` | 项目详情阶段状态下拉 | 从资源 bootstrap 动态加载 |
| 客户项目状态 | `customer_project_status` | 项目客户列表、客户详情关联项目状态 | 展示所有启用状态，并保留当前历史值 |
| 客户等级 | `customer_level` | 客户管理、资源客户维护、客户详情/列表展示 | 展示所有启用等级，并保留当前历史值 |
| 订单类型 | `order_type` | 订单新增/编辑、客户详情下单、订单统计 | 展示所有启用订单类型，并保留当前历史值 |
| 客户主状态 | `customer_master_status` | 客户管理、资源客户维护 | 从资源 bootstrap 动态加载 |
| 币种 | `currency` | 订单新增/编辑、统计换算目标 | 从资源 bootstrap 动态加载 |
| 销售状态/不可销售原因 | `salesability` / `unsellable_reason` | 新建项目、编辑项目、项目筛选 | 从资源 bootstrap 动态加载 |

## 本轮修复

- 修复任务详情状态下拉只显示“当前状态 + 允许流转状态”的问题。
- 新增通用配置选项函数：业务页面统一通过启用枚举生成选项，同时保留历史当前值，避免旧数据无法显示。
- 任务状态流转保留为“推荐流转提示”，不再阻止资源管理中启用的新状态被实际使用。

## 后续建议

1. 将当前 `prototype-runtime/app.js` 中的配置读取、表单校验、二次确认逻辑拆成独立模块，后续迁移到正式 React 页面时直接复用。
2. 后端接口层增加枚举合法性校验：前端可用只是体验层，最终仍应由后端校验枚举值是否启用。
3. 如果未来任务状态流转需要强约束，应增加“允许越级切换”或“管理员绕过”配置，不要再隐式隐藏已启用状态。
